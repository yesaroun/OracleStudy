2. 관리자 계정
- 모든 기능 사용 가능(교수자 계정 관리, 과정 관리, 과목 관리, 학생관리, 성적관리) - UI


3. 교수자 계정 관리 기능
- 관리자가 여러명 교수자 사전 등록 가능 - INSERT 
- 입력정보 : 교수자이름, 주민번호 뒷자리 (기본 패스워드) - 이름, 주민 번호(13자리) 입력 → 프로시저(아이디, 비번 생성) 즉 변수 4개 생성 이후 이를 바탕으로 INSERT (아이디는 자동으로 구분코드 + 번호 / 각각의 프로시저를 활용해서)
- 관리자는 모든 교수 정보 출력 기능 : 교수명, 과목명, 과목시간(시작, 끝), 교재명, 강의실, 강의 진행여부(강의 예정, 강의중, 강의종료) - SELECT

- 관리자는 교수자 정보 수정 삭제 가능 - FK인 경우 (교수 NULL 값 허용)(수강중에는 대체 교수가 있어야 삭제 가능(NOT NULL))

4. 과정 관리 기능 구현
- 관리자는 과정 사전 등록 가능
- 과정 정보 : 과정명, 기간(시작, 끝), 강의실 - INSERT (다만 강의실 DB를 먼저 입력 이후)

- 관리자는 모든 과정 정보 볼 수 있다 (구분코드, UI에서 다르다, 아이디 앞글자로 구분)
- 출력 정보 : 과정명, 강의실, 과목명, 기간, 교재명, 교수자명 -SELECT

- 입력된 과정 정보 수정 삭제 가능 - FK 고려, 예정인 경우만 삭제 가능



P3
5. 과목 관리 기능 구현
- 관리자 한 과정 당 여러개 과목 등록 가능
- 담당 교수 배정 가능
- 교수 변경 가능

- 과정 정보 : 과정명, 과목명, 과목기간(시작, 끝), 교재명, 교수자 명 (과정, 교수, 교재 존재해야 한다.) 없는 경우 에러 발생

- 출력 정보 : 과정명, 강의실, 과목명, 과목기간, 교재명, 교수자명

- 관리자는 과목 정보 수정, 삭제 가능 - 예정인 경우만 삭제 가능(과정 기간과 상관 없이)

- 총 배점은 100점 만점	- 배점 기능 INSERT

6. 학생 관리 기능 구현 (교수 정보 입력 프로시져 만드는 사람이 같이)
- 한개 과정당 여러 학생 등록 가능

- 학생 정보 : 이름, 주민번호 뒷자리(패스워드)

- 출력 정보 : 이름, 과정명, 수강과목, 수강과목 총점
- 중도 탈락의 경우 중도 탈락 사실 확인 가능 - SELECT 문에서 가능

- 학생정보 수정, 삭제 가능 - 성적 완료 이후 데이터 백업/ (조건 : 60일 이후 삭제 가능)

7. 성적 관리 기능 구현
- 과목 수강한 학생 성적 확인 가능
- 수강한 모든 과목 성적을 한페이지에서 확인가능 - SELECT (기간 이후는 백업 테이블 조회) 

- 중도 탈락의 경우도 이미 수강한 과목 정보 출력, 중도 탈락 사실 확인 가능

2 사용자측(교수자) 요구분석

1. 로그인 기능 구현	- SELECT 로 조건(과목이 끝난 수업만 서브쿼리)
- 로그인 이후 강의한 과목이 출력되고 성적 처리 화면으로 전환
- 배정된 강의중 진행되지 않은 과목은 성적 처리 화면으로 전환되지 않아야 한다.


2. 성적 입력기능
- 성적 입력 화면에서는 이름이 자동으로 입력되고 성적만 입력 하도록 한다 - SELECT로 학생 이름만 보여주고 INSERT로 성적 정보만

- 중도 탈락의 경우 성적 입력 화면에서 제외되어야 한다. - 조건절

- 배점은 교수자 결정
- 총 배점은 100점 만점	- (배점은 위에서 점수 입력만)
- 성적 입력시 출결, 실기, 필기 합이 화면에 자동 출력되어야 한다. - 입력 INSERT, 입력하면 바로 출력 되도록 프로시저
- 성적 입력후 저장 명령을 내리면 데이터베이스에 자동 저장 되어야 한다. - UI


3. 성적 출력 기능 구현
- 출력 정보 : 과목명, 기간, 교재명, 학생명, 출결, 실기, 필기, 총점, 등수

- 중도 탈락한 학생의 경우도 이미 수강한 과목에 대해서는 정보가 출력되어야 한다.
- 다만 중도 탈락 사실을 확인 가능해야 한다.
- 입력된 성적 정보 수정 삭제 가능	-- UPDATE,

3 사용자측(학생)

1. 로그인 기능 구현
- 로그인 이후 성적 확인 화면으로 전환

- 중도 탈락하여 명단에 제외되더라도 로그인 가능
- 다만 중도 탈락된 사실을 화면에서 확인 할 수 있어야 한다.

2. 성적 출력 기능 구현
- 이미 수강이 끝난 과목만 출력
- 수강 끝난 과목 선택시 성적확인 가능
- 출력 정보 : 이름, 과정명, 과목명, 교육기간, 교재명, 출결, 실기, 필기, 총점, 등수
- 본인 성적 정보만 출력	- ID나 수강코드 이용해서 조건
- 여러 과목이 끝난 경우 통합적으로 성적 볼 수 있어야 한다.
